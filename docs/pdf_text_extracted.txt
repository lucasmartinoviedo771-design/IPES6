PRESENTACIÓN DEL PROYECTO  
 
El presente proyecto propone SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE, una 
moderna aplicación web diseñada para unificar y facilitar la gestión administrativa y académica de 
nuestros estudiantes. 
Esta iniciativa surge de la experiencia directa en secretaría y bedelía, donde hemos identificado el 
esfuerzo y los riesgos de trabajar con información dispersa en planillas de cálculo, formularios de 
papel y sistemas que no se "hablan" entre sí. SIED SISTEMA INTEGRAL DE ESTUDIANTES Y 
DOCENTE resuelve esto: integra todos esos procesos en un único lugar, ofreciendo una plataforma 
accesible y confiable desde cualquier puesto de trabajo autorizado. 
Se presenta como una herramienta ya avanzada en su desarrollo, lista para ser el motor de la 
modernización de la gestión y el soporte clave para la toma de decisiones institucionales. 
 
FUNDAMENTACIÓN: 
 ¿Por qué necesitamos SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE? 
Diagnóstico de la situación actual 
Actualmente, la gestión de la información estudiantil en el instituto depende de: Planillas electrónicas 
(Excel), Formularios en papel y Sistemas parciales o no integrados. 
Este esquema, si bien funcional, genera problemas concretos que consumen tiempo y recursos: 
Duplicidad de Carga: El mismo dato debe ingresarse varias veces en distintos lugares. 
Riesgo de Errores: La transcripción manual inevitablemente lleva a datos desactualizados o con fallas. 
Demoras: Emitir una constancia o un informe consolidado toma más tiempo del necesario. 
Dificultad de Análisis: Es complejo obtener una visión completa de la trayectoria de un estudiante. 
Sobrecarga: El personal de secretaría y bedelía dedica valioso tiempo a tareas repetitivas. 
SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE es nuestra respuesta directa: centraliza 
la información en una única "fuente de la verdad" y automatiza las tareas rutinarias. 
Justificación del enfoque técnico elegido (Lenguaje sencillo) 
Hemos optado por desarrollar SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE como una 
"aplicación web completa" (técnicamente, una arquitectura full -stack), usando las siguientes 
herramientas:

TIPO DE PROGRAMA PROPUESTO: ¿Qué es SIED SISTEMA INTEGRAL DE ESTUDIANTES Y 
DOCENTE? 
La Interfaz del Usuario (Frontend) 
Denominación: SPA (Single Page Application). 
¿Qué significa? El sistema funciona como una aplicación moderna de teléfono o de escritorio, pero 
dentro de tu navegador. Al hacer clic en una sección, solo se actualiza esa parte de la pantalla, 
haciendo el sistema más rápido y ágil al usar. 
Herramientas clave: React y TypeScript. Esto asegura que la interfaz sea robusta y tenga menos 
errores. 
El Servidor Central (Backend) 
Función: Es el cerebro del sistema, donde reside la lógica para gestionar los legajos, calcular 
asistencias o verificar preinscripciones. 
Herramientas clave: Python y el framework Django. Son tecnologías de código abierto y ampliamente 
conocidas, lo que garantiza la estabilidad, la seguridad y la facilidad para realizar futuras mejoras sin 
depender de licencias costosas. 
El Archivador Digital (Base de datos) 
Motor principal: MySQL en contenedor Docker. 
Función: Motor de almacenamiento de datos confiable para este tipo de sistemas; se ejecuta aislado y configurable por variables de entorno.

Arquitectura por capas (El flujo de trabajo organizado) 
SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE está construido por capas. Piense en 
ello como una cocina profesional: cada persona (capa) tiene un trabajo claro y específico, lo que hace 
el proceso más eficiente y fácil de revisar. 
Capa de Datos (El Almacén): La base de datos MySQL, organizada para almacenar de forma eficiente. 
Capa de Negocio (El Chef): El código en Django que aplica las reglas del instituto (ej. si un estudiante 
puede o no inscribirse a una materia). 
Capa de Presentación (El Mesero): La interfaz en React que le muestra los datos al usuario en el 
navegador de una forma clara y fácil de entender. 
Escalabilidad y Seguridad (Preparados para el futuro) 
Seguridad: El acceso al sistema se realiza por roles (Secretaría, Bedelía, Rectorado), garantizando 
que cada usuario solo vea y edite lo que le corresponde. Usamos Tokens de Acceso (JWT) con fecha 
de expiración para una conexión segura.

ALCANCES DEL SOFTWARE: Qué tareas podemos automatizar  
SIED SISTEMA INTEGRAL DE ESTUDIANTES Y DOCENTE comienza resolviendo los puntos de

Áreas cubiertas (Módulos iniciales de impacto) 
Preinscripciones: 
Formulario web guiado y amigable para aspirantes. 
Generación automática de comprobantes y listados. 
Registro de la documentación pendiente por carrera. 
Gestión de Estudiantes (Legajos Centralizados): 
Alta, baja y modificación de datos desde un único punto. 
Asignación a carreras, cohortes y comisiones. 
Historial académico básico y unificado. 
Reportes e Informes Básicos: 
Generación instantánea de listados filtrados por carrera, cohorte o comisión. 
Capacidad de exportación a PDF y a Excel/CSV. 
Perspectivas de actualización y mejora futura (Una herramienta viva) 
El sistema está diseñado para crecer con el instituto, planificando a futuro: 
Ampliar el módulo de asistencia para toda la institución. 
Incorporar indicadores para monitorear la permanencia y el egreso. 
Integrar notificaciones automáticas (ej. correos a estudiantes o docentes). 
Facilitar la integración con sistemas de gestión educativa provinciales/jurisdiccionales.
